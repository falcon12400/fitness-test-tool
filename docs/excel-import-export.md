# Excel Import and Export Rules

## Design Goal

Excel should be familiar for users while the system still keeps a stable internal data format.

To achieve this, each exported Excel file contains:

- a visible worksheet for reading
- a hidden worksheet for system data

## Worksheet Structure

### Visible Worksheet: `Records`

Purpose:

- viewing
- printing
- backup inspection
- manual transfer between computers

Suggested columns:

- `Student Name`
- `Test Date`
- `Item 1`
- `Item 2`
- `Item 3`
- `Item 4`
- `Item 5`
- `Item 6`
- `Comment`

Recommended top note:

> This worksheet is for viewing and backup. Edit data in the web app for best compatibility.

### Hidden Worksheet: `_system`

Purpose:

- keep a stable machine-readable copy of the data
- support reliable re-import
- carry export metadata

Suggested fields:

- `schemaVersion`
- `exportedAt`
- `toolName`
- `toolVersion`
- `itemLabels`
- `recordsJson`

## Import Behavior for Version 1

- The system imports from `_system`
- The visible worksheet is not the official import source
- If `_system` is missing, the import should fail
- If `_system` is invalid, the import should fail
- The app should show a clear error when the file is not generated by the tool

## Why This Rule Is Used

- Users may accidentally add columns
- Users may rename headers
- Users may reorder columns
- Users may merge cells or break the table shape
- The first version should avoid fragile parsing rules

Using hidden JSON keeps import behavior predictable and reduces support risk.

## Recommended Error Messages

### Missing System Sheet

This Excel file does not contain the required system data and cannot be imported.

### Invalid System Data

The embedded system data is incomplete or invalid. Please use an Excel file exported by this tool.

### Unsupported File

This file format is not supported for import. Please import an Excel file generated by this tool.

## Future Relaxation Path

Later versions can gradually loosen the rules:

1. Detect whether the visible sheet differs from embedded JSON
2. Warn the user when a mismatch is found
3. Allow importing visible-sheet changes when the structure is still valid
4. Eventually support limited direct Excel editing for selected fields
