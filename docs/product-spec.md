# Product Spec

## Positioning

This is a lightweight web tool for managing student fitness test records.

The first-version rule is simple:

- the web app is the only official place to edit data
- Excel is used for viewing, backup, printing, and transfer

## Product Goals

- Manage a small number of student fitness test records
- Support adding, editing, and searching individual records
- Show a radar chart for one selected test record
- Export data to Excel for viewing and backup
- Re-import data from system-generated Excel files
- Keep the first version simple and easy to maintain

## Usage Assumptions

- Each test batch is small, roughly 25 students
- Each student has 6 test items per record
- Users mainly edit data in the web interface
- Excel is a supporting format, not the main editing interface

## Source-of-Truth Rules

- The web app data is the official source of truth
- Users must modify records through the web app
- Exported Excel files are for viewing, backup, printing, and transfer
- When importing an Excel file, the system reads embedded JSON data
- Manual edits to visible Excel sheets are not guaranteed to be applied

## In-Scope Features

Version 1 should include:

- record list
- create record
- edit record
- student search
- radar chart view
- Excel export
- Excel import

## Out-of-Scope Features

Version 1 should not include:

- direct Excel editing as an official workflow
- batch editing for a full test column
- multi-user collaboration
- cloud sync
- backend database
- PDF export
- multi-class management

## Suggested Pages

### Record List

- Display all student records
- Support searching by student name
- Allow selecting a record for editing or analysis

### Record Editor

- Create or edit a single fitness test record
- Fields include student name, test date, 6 test items, and comment

### Radar Analysis

- Display a radar chart for one selected record
- Help users quickly review score distribution across all test items

### Import and Export

- Export Excel
- Import Excel
- Show usage rules and warnings

## Data Fields

Each record should include:

- `id`
- `studentName`
- `testDate`
- `item1`
- `item2`
- `item3`
- `item4`
- `item5`
- `item6`
- `comment`

The app should also reserve configurable labels for the six items:

- `itemLabels`

Example:

- `["Flexibility", "Balance", "Core", "Power", "Agility", "Coordination"]`

## Suggested Storage Strategy

- Internal browser storage: `localStorage` or `IndexedDB`
- Exchange format: Excel (`.xlsx`)
- Embedded backup inside Excel: hidden JSON sheet

## Import Rule

- Import only guarantees compatibility with Excel files generated by this tool
- The app reads hidden system data first
- The visible sheet is not treated as the official editable data source in version 1

## User-Facing Message

Recommended message for the import/export page:

> Exported Excel files are intended for viewing, backup, printing, and transfer. To avoid import failures or inconsistent data, please edit records in the web app instead of editing the Excel file directly.

## Future Expansion

Later phases may add:

- conflict detection between visible sheets and embedded JSON
- limited Excel-editable fields
- batch editing for one test item
- before/after comparison for one student
- multi-class management
